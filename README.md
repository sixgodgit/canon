# Hermes 技能生态图书管理员

> 不是刽子手，是园丁。不是审判官，是图书管理员。

## 项目简介

本项目记录了 Hermes Agent 技能生态管理系统的设计演进、核心文档和实现代码。

## 历史演进

### v1.0: skills-judgment（审判系统）
- **时间**：2026年4月
- **定位**：基于自然选择的技能生命周期管理
- **特点**：
  - 六维评分体系
  - 自动清理低分技能
  - 缺乏人性化考量

### v2.0: hermes-skill-curation（园丁系统）
- **时间**：2026年5月
- **定位**：系统化评估、分类、维护技能库
- **改进**：
  - 增加反熵保护机制
  - 对抗性辩护层
  - 人类在回路
  - 核心原则：不是刽子手，是园丁

### v3.0: skill-ecosystem-librarian（图书管理员）
- **时间**：2026年6月
- **定位**：四层架构技能管理
- **架构**：
  - 热气层（Hot）：高频使用
  - 常驻层（Normal）：稳定使用
  - 种子库（Seed Bank）：潜在价值
  - 隔离层（Isolation）：待评估

## 核心原则

1. **图书管理员不烧书** - 保留所有技能，分类管理
2. **园丁不灭种** - 保护低频但有价值的技能
3. **审计员不执刑** - 只建议，不删除

## 使用方法

### 评估技能
```bash
# 加载园丁技能
skill_view(name='hermes-skill-curation')

# 执行评估
# 按照 SKILL.md 中的流程执行
```

### 四层分类
- **热气层**：最近7天内被调用
- **常驻层**：30天内被调用
- **种子库**：有潜在价值但未使用
- **隔离层**：待评估或问题技能
