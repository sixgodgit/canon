# Hermes 技能生态管理系统演进历史

## 概述

Hermes Agent 的技能生态系统经历了三个主要阶段的演进，从简单的审判系统发展为完整的图书管理系统。

## v1.0: skills-judgment（审判系统）

### 时间线
- **创建时间**：2026年4月
- **核心文件**：`skills-judgment/SKILL.md`

### 设计理念
基于自然选择的技能生命周期管理，通过评分机制自动清理低质量技能。

### 核心功能
1. **六维评分体系**
   - 活跃度：最近调用频率
   - 关系度：被其他技能引用次数
   - 完整度：文档字段完整性
   - 稳健度：执行成功率
   - 适应度：与系统兼容性
   - 独特性：不可替代价值

2. **判决规则**
   - 70-100分：保留
   - 50-69分：观察
   - 30-49分：缓刑
   - 20-29分：冷藏
   - <20分：建议删除

### 局限性
- 缺乏人性化考量
- 可能误删有价值技能
- 没有反熵保护机制

## v2.0: hermes-skill-curation（园丁系统）

### 时间线
- **创建时间**：2026年5月
- **核心文件**：`hermes-skill-curation/SKILL.md`

### 设计理念
从"审判"转变为"园丁"，目标是维护生态健康，不是最小化技能数。

### 核心改进

#### 1. 反熵保护机制
- 每轮评估随机保护5-10%的低分技能
- 理由：今天没用的技能，明天环境变了可能突然重要
- 优先保护：创意类、游戏类、ML研究类、智能家居类

#### 2. 对抗性辩护
- 对每个拟删除技能，必须先写"为什么不该删"
- 回答5个问题：
  1. 什么场景有用？
  2. 未来会复活吗？
  3. 代表什么独特价值？
  4. 有用户依赖吗？
  5. 删除后有什么不可逆损失？

#### 3. 人类审批
- 所有删除必须用户批准
- 系统只建议不执行

### 核心原则
- **你不是刽子手，是园丁**
- **保护多样性**
- **对抗性辩护**
- **人类审批**

## v3.0: skill-ecosystem-librarian（图书管理员）

### 时间线
- **创建时间**：2026年6月
- **核心文件**：`skill-ecosystem-librarian/SKILL.md`

### 设计理念
从"园丁"升级为"图书管理员"，建立四层架构的分类管理系统。

### 四层架构

#### 1. 热气层（Hot）
- **定义**：最近7天内被调用的技能
- **特点**：高频使用，核心功能
- **管理**：自动维护，优先更新

#### 2. 常驻层（Normal）
- **定义**：30天内被调用的技能
- **特点**：稳定使用，功能完整
- **管理**：定期检查，适时优化

#### 3. 种子库（Seed Bank）
- **定义**：有潜在价值但未使用的技能
- **特点**：低频但独特，可能未来重要
- **管理**：保留可能性，不主动删除

#### 4. 隔离层（Isolation）
- **定义**：待评估或有问题的技能
- **特点**：可能损坏、重复或过时
- **管理**：隔离观察，人工审核

### 核心原则
1. **图书管理员不烧书** - 保留所有技能，分类管理
2. **园丁不灭种** - 保护低频但有价值的技能
3. **审计员不执刑** - 只建议，不删除

## 演进对比

| 特性 | v1.0 审判 | v2.0 园丁 | v3.0 图书管理员 |
|------|-----------|-----------|-----------------|
| **定位** | 清理者 | 维护者 | 管理者 |
| **目标** | 最小化技能数 | 维护生态健康 | 分类组织管理 |
| **删除策略** | 自动删除低分 | 建议删除+人工审批 | 分类隔离不删除 |
| **保护机制** | 无 | 反熵保护 | 四层架构保护 |
| **辩护机制** | 无 | 对抗性辩护 | 价值评估 |
| **架构** | 扁平 | 扁平+保护 | 四层架构 |

## 未来展望

1. **自动化程度提升** - 更多自动分类和维护
2. **智能推荐** - 基于使用模式推荐技能
3. **生态健康度指标** - 量化评估生态健康
4. **跨平台同步** - 多实例技能同步
